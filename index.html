ABOUT

How I think and build

I’m at my best when there’s no clear playbook: I research standards, define requirements, test deliberately, and communicate decisions so teams can execute confidently—with care for the people using the system.

What I do as an engineer

  • Power distribution planning: loads, headroom, component ratings, and protection
  • Safety-first wiring: fusing strategy, switching, grounding approach, documentation
  • PCB + prototyping: breadboard → validation → iteration
  • Testing mindset: staged checks, failure modes, and clear acceptance criteria
  • Communication: explain tradeoffs, align stakeholders, ship responsibly

Multi-faceted profile

  • Engineering rigor + calm, clear communication
  • Project coordination from leadership roles (USGC lead coach, mentorship)
  • Empathy-driven: design for safety, usability, and real people
  • Self-directed learning: I build skill fast when faced with unfamiliar problems
